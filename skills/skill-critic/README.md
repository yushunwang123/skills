# 你的 AI 正在批量生产"看起来很有用"的废物 Skill

—— 认识一下 `skill-critic`，专治这个病。

---

## 一个尴尬的真相

自从 Agent Skill 火了，每个人都在疯狂造 Skill。但说句实话——**大部分新建的 Skill，根本不该存在。**

它们有三种死法：

- **从没被触发过** —— 描述写得像废话，AI 永远想不起来用它；
- **大而无当** —— 一个 Skill 想干 PDF、Excel、发邮件三件事，结果一件都干不好；
- **纯属复读** —— 里面写的东西模型本来就会，纯属占用上下文、浪费 token。

问题是：**写的时候你根本看不出来。** 每个 Skill 在作者眼里都闪闪发光。

缺的不是"写 Skill 的工具"，而是一个**敢说真话的审稿人**。

---

## `skill-critic`：给 Skill 请一位毒舌评审

它自己**一行 Skill 都不写**。它只做一件事——**审**。

在你动手写之前，它审你的"计划"；写完之后，它审你的"草稿"。同一套标准，六个维度，逐项打分 **Pass / Weak / Fail**，每条都甩给你最该改的那一刀：

| # | 它逼你回答的问题 |
|---|---|
| 1 | **该不该建？** 这事值得做成 Skill 吗，还是一句 prompt 就够了？ |
| 2 | **边界清楚吗？** 一句话说得清范围吗，还是个什么都装的杂物箱？ |
| 3 | **有干货吗？** 把模型本来就会的删光，还剩多少？ |
| 4 | **能照着做吗？** 只看说明、不靠猜，能跑通吗？ |
| 5 | **说清"别做什么"了吗？** 还是只会教"要做什么"？ |
| 6 | **结果可验证吗？** 有没有能检查对错的标准，而不是"反正你看着办"？ |

最后吐给你一张**结论表**：一个明确的 verdict（建 / 别建 / 合并 / 发布 / 返工）+ 六维状态 + 必修清单。规矩很硬——**Fail 直接拦截，Weak 必须写明理由才放行。** 没有和稀泥。

它甚至防着审稿人自己犯浑：不许橡皮图章式放行、不许在错别字上挑刺却放过真问题、不许强加用户没要的功能、不许把"我会换种写法"当成缺陷。

---

## 它不是空谈——它把自己审到改了名

最好的广告是真实的故事。

这个 Skill 每出一版，我们都干同一件狠事：**拿它自己的六维规则，去审它自己。**

第一轮就翻车——它揪出自己两处"双标"：要求别人写"非目标"，自己却没写；要求别人列"反模式"，自己只列了给别人挑的、没列审稿人自己的。改。

以为过关了？再审一轮，它对自己下了更狠的一刀——**连自己的名字都没放过**：原名 `skill-reflection`（"反思"）配不上它"毒舌、敢拦"的人设，也卡不住"评 / 审 / 拦"这些真实触发词。于是它把自己改名成了 `skill-critic`，顺手把 README 里那句心虚的"六维全过"也变成了真的。

**一个能把自己从内容一路审到名字、亲手逼自己改名的 Skill —— 这套标准不是摆设，是真能咬人，连主人都咬。**

## "这不就是 skill-judge 吗？"

会有人这么问，很正常。诚实说：**内核确实撞车。** "知识增量是核心维度""反模式要显式写""结果要可验证"——这几条 skill-judge 也有，装不了独家。

但它故意做了三件 skill-judge 没认真做的事：

| | skill-judge | skill-critic |
|---|---|---|
| **什么时候用** | 事后审一份**已经写完**的 SKILL.md | 你**动手写之前**先审"计划"，写完再审"草稿" |
| **吐什么结果** | 打分 + 等级（120 分 / A–F） | 一个**决策**：建 / **别建** / **合并进 X** / 发布 / 返工 |
| **第一刀砍哪** | 默认"你已经决定要建了"，只评好坏 | 第一维直接问"**该不该存在**"——当场劝退，或让你去改隔壁那个 |

一句话区别：**skill-judge 是验收员，它是立项评审 + 验收员。** 它最值钱的不是评分多花哨，而是**早一步**、而且**敢说"别建"**——在你连大纲都没写的时候，就告诉你"这事一句 prompt 就够了"。

所以它不是 skill-judge 的劣化版，也不是颠覆者：**要严格量化打分、给团队定 A/B/C 门槛，用 skill-judge；要前置劝退、决策导向、随手就能跑，用它。** 两件不同的事。

---

## 谁需要它？

- 你准备建一个新 Skill —— 先让它审计划，省得白写；
- 你 Skill 库越攒越多、质量参差 —— 拿它逐个体检；
- 你想给团队定一条"什么样的 Skill 才能合入"的硬门槛 —— 它就是那把尺子。

它是 `create-skill`（负责**写**）的黄金搭档：**一个负责造，一个负责拦。** 造得快不稀奇，拦得住才值钱。

---

## 怎么用

不用记命令。把下面任意一种东西丢给它，说一句"**用 skill-critic 审一下**"就行：

- 一个**还没动手的想法**（"我想建个 X Skill"）→ 它先帮你判该不该建；
- 一份**写到一半的草稿** → 它挑出致命伤再让你继续；
- 一个**现成的 `SKILL.md`** → 它逐维体检、给返工清单。

它会回你一张**六维结论表 + 一个 verdict**。要动手写的时候，接力交给 `create-skill`——**一个拦，一个造。**
